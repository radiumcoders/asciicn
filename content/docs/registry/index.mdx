---
title: Registry setup
description: How to author and distribute components with a shadcn registry.
---

This project is structured as a **shadcn source registry**. You keep real
component source under `components/asciicn`, generate `registry.json`, then
build installable JSON payloads into `public/r`.

## Layout

```txt
components/asciicn/     # component source (what you edit)
registry.json           # generated source registry (items + file paths)
public/r/               # built registry JSON for the CLI
scripts/build-registry.mjs
```

## 1. Add a component

Create a file (or folder) under `components/asciicn`:

```tsx title="components/asciicn/ascii-badge.tsx"
import { cn } from "@/lib/utils"

export function AsciiBadge({ className, ...props }: React.ComponentProps<"span">) {
  return (
    <span
      className={cn(
        "inline-flex border border-border px-1.5 py-0.5 font-mono text-xs",
        className
      )}
      {...props}
    />
  )
}
```

Folder form also works:

```txt
components/asciicn/my-widget/
  my-widget.tsx
  styles.css
```

The build script groups files by component name and writes one registry item
per group.

## 2. Generate registry JSON

```bash
pnpm registry
```

This runs `scripts/build-registry.mjs`, which:

1. Scans `components/asciicn`
2. Writes root `registry.json` (source registry)
3. Runs `shadcn build` → `public/r/*.json` (built registry)

`pnpm dev` and `pnpm build` run the same step first so the registry stays in
sync while you work.

## 3. Source registry shape

Generated `registry.json` looks like:

```json
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "name": "asciicn",
  "homepage": "https://asciicn.local/docs",
  "items": [
    {
      "name": "ascii-badge",
      "type": "registry:ui",
      "title": "Ascii Badge",
      "description": "ASCII-styled Ascii Badge component from asciicn.",
      "files": [
        {
          "path": "components/asciicn/ascii-badge.tsx",
          "type": "registry:ui",
          "target": "components/asciicn/ascii-badge.tsx"
        }
      ]
    }
  ]
}
```

`registryDependencies` are inferred when a file imports from
`@/components/ui/*` (official shadcn UI pieces).

## 4. Host the built files

Serve `public/r` from your site (this Next app already does that at `/r/...`).

| File | URL |
| --- | --- |
| Index | `/r/registry.json` |
| Item | `/r/ascii-badge.json` |

Point consumers at your origin, for example:

```txt
https://your-domain.com/r/{name}.json
```

## 5. Consumer components.json

In a project that wants to install from this registry:

```json
{
  "registries": {
    "@asciicn": "https://your-domain.com/r/{name}.json"
  }
}
```

Then:

```bash
pnpm dlx shadcn@latest add @asciicn/ascii-badge
```

For local testing without a public URL:

```bash
pnpm dlx shadcn@latest add ./public/r/ascii-badge.json
```

## 6. GitHub source registry (optional)

A public GitHub repo with root `registry.json` can be used as a **source**
registry without hosting `public/r`:

```bash
pnpm dlx shadcn@latest add owner/repo/ascii-badge
```

This project still builds `public/r` so you can serve a classic URL registry
from the docs site.

## Scripts

| Command | What it does |
| --- | --- |
| `pnpm registry` | Scan components + build JSON |
| `pnpm dev` | Registry build, then Next dev |
| `pnpm build` | Registry build, then Next production build |

## Tips

- Keep distributed components free of app-only imports
- Prefer `@/components/ui/*` for shared primitives so deps resolve via shadcn
- Re-run `pnpm registry` after adding or renaming files if you are not on `dev`
